# track-state-machine -- final champion snapshot

This is the promoted output of the `track-state-machine` Track (loop-02),
one of two competing architecture Tracks in the messy-vendor-CSV parser
experiment. It implements a hand-rolled **whole-input character state
machine** (see `parser.py` module docstring for the full design) as an
alternative to a line-oriented/regular-expression approach.

## Files

- `parser.py` -- the public API: `ParseError` and
  `parse(text: str) -> list[list[str]]`. No `csv`, `pandas`, or other
  third-party parsing library is imported.
- `test_parser.py` -- 53 unittest cases covering every requirement in the
  Track prompt (BOM, comments, blanks, quoted commas, embedded LF/CRLF,
  doubled-quote escaping, ragged rows, trimming vs. preservation,
  comment/blank look-alikes retained inside quotes, empty fields/trailing
  commas, CR-only physical lines, unicode content, and malformed-input
  cases asserting exact line/column/row/field). Run with:

  ```
  python test_parser.py
  ```

- `fixtures/sample.csv` -- a small, valid, deliberately messy example file
  exercising a BOM, a comment line, a blank line, a quoted field with an
  embedded comma, an embedded newline, a doubled-quote escape, leading/
  trailing whitespace around an unquoted field, and a ragged row.
- `fixtures/malformed_unterminated_quote.csv`,
  `fixtures/malformed_quote_in_unquoted.csv` -- two malformed examples used
  to demonstrate actionable `ParseError` messages.
- `demo_fixtures.py` -- optional manual demo script (not part of the
  objective test suite) that parses the fixtures above and prints the
  result, including the two `ParseError` messages.
